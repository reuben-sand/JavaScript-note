1.developer tool中的CSS部分

在開發者工具中，可查看指定元素的CSS設定，每個瀏覽器的位置不一定一樣，在chrome中的element tab中，左半部分是DOM tree，可指定元素，右半部分的style tab即可查看，影響該元素的所有選擇器，若旁邊標註user agent stylesheet，意指預設樣式。

2.CSS導入HTML檔案的方式

CSS導入HTML檔案的方式有三種類型，external、internal、inline。

a.external method

該類型導入方式有兩種:

第一種，在head元素中，使用<link rel="stylesheet" href="CSS檔案路徑">
第二種，在head元素中，使用<style>@import url('CSS檔案路徑');<style>

b.inline method

該類型導入方式，直接在要導入風格的html元素中加入style屬性，在屬性內放置將要應用在該元素中的CSS屬性和值對，如:<p style="color:red;" >paragraph</p>

c.internal method

該類型導入方式，在head元素中，直接放入選擇器，如:

<style>
            selector{
                property: value;
            }
        </style>

3.optimizing image and retina display

為了支援高像素濃度的螢幕，如:retina screen，設置圖像時，可以選擇實際渲染的尺寸兩倍、三倍大的檔案。

4.絕對路徑和相對路徑

連結檔案的路徑分成絕對路徑和相對路徑。

絕對路徑

不管使用該路徑的檔案在哪裡，該路徑引入的檔案都是一樣的，絕對路徑開頭會加上http or https，若img的url是使用絕對路徑連結到他人網站的檔案(被稱為hotlink)，未經允許消耗他人網站的頻寬(當訪客打開我們的網頁時，嵌入的資源會直接從原始伺服器載入)，以及有可能侵犯版權，需要避免。

相對路徑

根據使用該路徑的檔案，該路徑引入的檔案都是可能不一樣，若開頭為/被稱作根相對路徑，開頭的/代表根，以根為出發點去找尋要引入的檔案的路徑。

9.定義顏色相關屬性的值類型

顏色相關屬性值可以是identifiers，也可以是Hexadecimal或function rgb、function hsl等。

color identifiers

基本的color identifiers如:white、blacks、blue、yellow、red等。進階的color identifiers，如:salmon、khaki、coral、beige等。

color Hexadecimal

color值也可以用#+6個16進位的數字表示。按照RGB字母的順序，前兩個數字代表R、兩個數字代表G、最後兩個數字代表B，若色光值兩個數字一樣，可簡寫，如:#FFFFFF(white)、#000(black)。

rgb function

color值可以用rgb function，rgb()的引數需要有R、G、B三個數值，數值可是0~255或是0~100%，若需要設定透明值可使用rgba()，只是多一個alpha值(不透明值)，該值可是0~1或0~100%。如:rgba(255,0,0,.5)。

hsl function

color值可以用hsl function，hsl()的引數需要有Hue、saturation、lightness三個數值，Hue值範圍為0~360deg(不加單位預設deg，也可用其他angle值)，saturation、lightness值使用0~100%，若需要設定透明值可使用hsla()，只是多一個alpha值(不透明值)，該值可是0~1或0~100%。如:hsla(.5turn,50%,50%,1)。

10.類型選擇器和通用選擇器

type selector用來定義所有該類型元素的樣式，語法直接輸入該html的元素名稱後面接宣告塊，而universal selector用來定義所有元素的樣式，語法上以*代表後面接宣告塊。 

        p{
            color: red ;  
        }
        *{
            color: red ;  
        }

11.class選擇器和id選擇器

class selector選取特定class的元素，語法是.+要選取的class值後面接著宣告塊，id selector選取特定id的元素，語法是#+要選取的id值後面接著宣告塊。id選擇器的specificity大於class選擇器大於類型選擇器。 

        .className{
            color: red ;  
        }
        #idName{
            color: red ;  
        }

Compound Class Selector

Compound Class Selector指定一個以上特定的class選擇，包含所有指定class的元素才能被選取，語法為兩個以上class選擇器緊接著。type selector搭配class selector也可以用類似用法。如下: 

        .className.className2{
            color: red ;  
        }
        p.className{
            color: red ;  
        }

12.descendant選擇器和group選擇器

descendant選擇器屬於Combinator selector用來選擇符合特定嵌套關係的元素，藉由兩個以上選擇器名稱中使用空格分開，再接上宣告塊，左邊的選擇器為ancestor，右邊的選擇器為descendant，關係不用是parent-child(可跳層)。group選擇器是讓不同選擇器共用同一宣告塊，不同選擇器名稱間用逗號分隔。兩者示範如: 

        main section p{
            color: red ;  
        }
        h1,h2{
            color: red ;  
        }

13.inheritance and specificity

inheritance

如果property具有inheritance，若某個元素擁有該property，子元素也會繼承。

specificity

specificity決定選擇器的優先權，選擇器的specificity從小到大排列為:繼承的property、universal、type(pseudo-element)、class(pseudo-class)、id selector。若是combinator需要考慮選擇器的數量，先比較id選擇器的數量，若一樣再比較class選擇器數量，最後才是比較類型選擇器數量。若specificity相同以後面出現的選擇器為主。universal selector不影響specificity。

14.!important and pseudo-class selector

!import declaration

除了specificity和code順序，內聯CSS有更高的優先權，而有!import declaration的property又更優先，!import宣告用法為，將該宣告附加在property value pair後面。b如: 

        p{
            color:red !import;
        }

pseudo-class selector 1

pseudo-class selector需要與其他selector連結，使用":"在其他選擇器的後面連接，這裡介紹五個pseudo-class selector，分別是link、visited、focus、hover、active。link、visited需要用在含有href屬性的anchor元素，前者代表未拜訪過的連結(預設藍色)，後者是已拜訪過的連結(預設紫色)，使用時普通的anchor類型選擇器要注意覆蓋預設樣式。示範如: 

        <a class="link11" href="#chapter11">通往第十一章</a>
        
        .link11:link{
            color:red;
        }
        .link11:visited{
            color:beige;
        }
通往第十一章

pseudo-class selector 2

另外三種pseudo-class selector中，focus代表元素在focus狀態，該狀態的觸發是使用滑鼠點擊or tabs or tab key造成。hover則是鼠標在該元素上方觸發。active所應用的元素是需要由用戶啟動的元素，當啟動時(例如按鈕點擊期間)觸發。使用時也要注意覆蓋問題，如以下示範: 

        <button>abcd</button>
        button:active{
            color:purple;
        }
        button:hover{
            color:white;
        }
        button:focus{
            background:white;
        }

15.inline element and block element

inline element的高和寬等同content，高和寬無法藉由設定高度和寬度property改變，padding和margin上和下無法影響顯示，inline元素間並排顯示，直到空間不夠才換行；而block element高度同內容，而寬度橫跨它所在的容器，元素之間堆疊排列。屬於inline or block element雖然是預設的，但是可以透過display:inline or block，改變元素行為，也可設定display:inline-block，讓元素能並排顯示，且可設定高度、寬度、padding、margin。

16.box model

box model包含四個部分，content、padding、border、margin，content及元素包含的內容，根據元素類型可能被width和height影響，padding代表在元素內包圍content的空間，margin代表圍繞元素的空間，border是介於padding和margin之間的邊界。

padding

padding property is shorthand，包含padding-top padding-right padding-bottom padding-left的值，只能是正的數值，當padding只有一個值時，代表上下左右等同該值，只有兩個值時，第一個值代表上下，第二個值代表左右，當有三個值時，分別代表上、左右、下，當有四個值時，按照順時針分別代表上、右、下、左。

margin

margin property is shorthand，包含margin-top margin-right margin-bottom margin-left的值，可以是正或負的數值，當margin只有一個值時，代表上下左右等同該值，只有兩個值時，第一個值代表上下，第二個值代表左右，當有三個值時，分別代表上、左右、下，當有四個值時，按照順時針分別代表上、右、下、左。

border

border property is shorthand，包含border-width border-style border-color，可不按照順序，也可只指派其中一部分值，按照此順序去排列shorthand的值。如以下示範: 

        <span>abcd</span>
        span{
            border:5px dotted white;
        }

17.margin collapse and align

margin collapse是CSS的一個重要概念，主要發生在block element的垂直邊距，兩個不同元素的margin合併，保留較大的值，有三種情況，第一種是相鄰的塊狀元素，上面的元素的下邊距與下面的元素的上邊距。第二種狀況是父元素和子元素的同位置邊距合併。第三種情況是空元素的邊距合併，空元素自己的上下邊距會合併。要解決邊距合併有三種方式，第一是在父元素、相鄰元素、空元素會合併邊距的位置加上border或padding。第二是使用float，第三是使用flexbox or grid。

使用margin對齊水平中心

可以設定margin使元素對齊，只需要將margin-right and margin-left設置為auto。

18.font-family

font-family是用來指定元素使用字體的一個優先清單，可指定多個字體名稱或是通用名稱，以逗號分隔，最左方的字體優先，建議最右方放置一個通用名稱，字體名稱可以是字串或是identifiers﹐若是包含連字號外的標題符號、數字、空格，最好使用字串值，若使用identifiers需要使用\在特殊字符(空格不能)前方轉義。

常用的generic-name

常用的generic-name有serif,sans-serif,monospace,cursive,fantasy。serif是指字母中一條小線或筆觸，附加在較大的筆觸中，在serif字體中，字型的筆畫末端呈現喇叭型或錐形。sans-serif字體是指沒有serifs的字體，字型的筆畫結尾簡單沒有裝飾。monospace字體指所有字形都有相同的固定寬度，通常用於code。cursive字體，通常具有連接的筆畫、以及超出斜體範圍的草寫特徵，呈現出的效果像是手寫，而非印刷字體。fantasy主要是包含有趣字元的裝飾字體。cursive和fantasy通常用於標題和小部分的文本、以及裝飾。

web safe font

web safe font指的是一組在大多數操作系統和瀏覽器上都能夠正確顯示的字體。這些字體因其廣泛的可用性和一致的外觀，通常被用於網頁設計。使用這些字體可以減少因為用戶系統中缺少某些字體而導致的顯示問題。

19.font-weight and font-style

font-weight

font-weight是用來指定相同字體的不同weight，但需要依賴該字體組有什麼weight上的選擇。font-weight的值可以是數值或identifiers，數值可以是1~1000，數值越高表示越粗，數值400對應identifier normal，而700對應identifier bold。identifiers還有bolder和lighter，根據從父元素繼承值選擇較粗或較細的字體weight。如以下表格:

font-style

font-style用來設定font是否為斜體樣式，值可以是italic，normal，oblique，oblique+<angle>。italic將選擇分類為italic類型的字體，若沒有italic類型將選擇oblique，normal將選擇分類為normal類型的字體，oblique將選擇分類為oblique類型的字體，若沒有oblique類型將選擇italic，oblique還可以加上指定傾斜角度(預設14度，-90 度到 90 度)，若oblique和italic類型找不到對應的字體，可能會使用人工模擬。使用italic or oblique，且第一個字體沒有內建斜體時，瀏覽器會模擬斜體效果，而不是自動切換到第二個字體的原生斜體版本。如以下示範:

20.@font-face

@font-face是一種CSS @規則，用來指定用於展示文本的定義字體，字體能夠從遠端伺服器裝載，或在使用者電腦上已安裝的本地字體取的。在該規則的區域塊內，使用font-family property去自定體該字體被使用時的名稱。並使用src property定義該字體檔案路徑(可多個路徑)，其值為url()和format() function，如以下示範: 

        @font-face{
            font-family:"Reuben Font";
            src:url('檔案路徑') format('woff2'),
                url('檔案路徑2') format('woff');
        }
        h1{
            font-family:'Reuben Font';
        }

web font

Web font是指專門為網頁使用而設計或調整的字體，這些字體是通過網頁的 @font-face 規則動態加載到用戶的瀏覽器中顯示的。常用的提供web fonts平台有Google Fonts、Adobe Fonts(部分需要訂閱)。網頁開發者只需要複製簡單的平台提供的code即可使用，不需要手動輸入@font-face設置。

21.font-size

font-size是用來定義字體大小的property，更改字體大小可能影響到在字體相關property中，使用相對長度單位的後代元素。一些元素有預設的font-size，如small元素的字體大小為相對父元素字體的80%，即0.8em，h1元素的字體尺寸為2em or 32px，p元素的字體大小為1em or 16px，code和pre元素的字體大小為13px(除了firefox)。

字體尺寸常用單位

字體尺寸常用的絕對單位為px(pixel)，常用的相對單位為em、rem，1em等於父元素的字體尺寸，1rem等於根元素<html>的字體大小(預設16px)。字體尺寸設定如以下示範: 

        <p>預設大小</p>
        <p style="font-size:24px;">24px</p>
        <p style="font-size:2em;">2em</p>

22.font property

font property是簡寫，包含font-style、font-weight、font-variant、line-height、font-size、font-family，使用簡寫時font-size和font-family是必要的，line-height需要使用/符號接在font-size後，font-family需要在最後的位置，font-variant、font-style、font-weight需要在font-size前面。

line-height

line-height property在水平書寫的模式下，設置line box的高度，在垂直書寫的模式下，設置line box的寬度，值可以是identifiers normal，也可以是無單位的數值(*font-size)，或<length>或<percentage>單位，通常建議使用無單位數值，因為其子元素也會繼承該倍數，若是使用em或%將計算完後才讓子元素繼承。如以下示範

        </div>
        <div class="lh-test-unitless">
            <h1>Avoid unexpected results by using unitless line-height.</h1>
            Length and percentage line-heights have poor inheritance behavior.
        </div>
        
        .lh-test-em{
            line-height:1.2em;
        }

28.flexbox

flexbox是一種模組，用於排列item在同一軸上，使用方式為在item的容器上使用，display:flex(該容器成為block元素)，或是display:inline-flex(該容器成為inline元素)，接著該容器將成為flex-container，而子元素將成為flex-item，item寬度會等同內容，item高度會等於所有items裡最高的content高度，flex-conatiner尺寸根據display的設定，按造inline元素或block元素的模式。

flex-container setting

在預設上main-axis是水平的，代表item的排列方向，而cross-axis是垂直於main-axis，會影響換行方向，可藉由在容器上使用flex-direction改變main-axis方向，該值有row(預設，文字排列方向)、row-reverse(與row相反)、column(從上到下)、column-reverse(從下到上)。flex-wrap property能夠設定元素是否換行(當item總寬度超出容器寬度後，且flex-shrink:0;)，以及換行方向(cross-axis方向)，其值有no-wrap(預設)，wrap(上到下)，wrap-reverse(與wrap反向)。flex-direction和flex-wrap，可簡寫成flex-flow。

flex-item setting

flex-item有三種property可以設置不同狀況的item size，分別是flex-grow、flex-shrink、flex-basis，flex-grow的值為0(預設)、1、更高的數值，非0時代表當容器有空間時，會擴大元素填滿空間，而數值越大代表該元素填滿的比例越重。flex-shrink的值為0、1(預設)、更高的數值，非0時代表當items超出該容器，將縮減所有item的尺寸以符合容器，數值高代表縮減的比例高。flex-basis代表該item沒有特別狀況時的尺寸，會覆蓋寬度或高度，上述item的尺寸指主軸方向上的尺寸。上述提到三個屬性可以使用flex當作簡寫，其值按照順序應為，flex-grow、flex-shrink、flex-basis，如以下示範:

29.grid

grid是一個CSS模組，可將頁面劃分為主要區域，或定義 HTML的元素構建的控件各部分之間的關聯，就尺寸和位置和層級而言。grid layout是一種二維佈局，對齊item在rows and cols。與flexbox差別在於grid佈局較關注父元素。要使用grid模組，需在父元素上設定display:grid，此時父元素會成為grid-container、子元素會成為grid-item。

grid結構

grid line是劃分cell or track的垂直線或水平線， 用於定位grid-item，可用正負index(grid最左邊或最頂邊是1)來指定grid column(row)line。grid cell(兩條相鄰垂直和兩條相鄰水平grid line之間的空間)是網格的最小單位，用來放置grid-item，grid track是兩條相鄰grid-line之間的通道，水平的是grid row，垂直的是grid-column。grid area是由任意數量cell組成的空間，grid gap(gutter)是相鄰track間的間距。

grid-container

要讓item在網格上排列前，需要設置網格的大小、數量，網格有分為顯示網格和隱式網格(有需要才顯示)。設置顯示網格需在網格容器上使用grid-template-columns、grid-template-rows property，可包含多個值，值之間需用空格分隔，其值可以是<length>值或fr值(表示一個分段比例)，也可以搭配repeat()function 來表示多個值，repeat()funtcion 可輸入兩個引數，第一個引數用來表示重複的數量，第二個引數用來表示重複的值。設置隱式網格使用grid-auto-columns和grid-auto-rows property其值類型與顯示網格的值一樣。要定義track之間的距離使用column-gap and row-gap property，可簡寫成gap property，第一個值表示row-gap，第二個值表示column-gap，值類型為<length>。

grid-item

若要設定grid-item排列在網格的位置，佔據的cell數量，可在grid-item中使用grid-column-start、grid-column-end、grid-row-start、grid-row-end，表示item佔據的grid cell組成的grid line，其值為代表起始或結束的水平或垂直grid-line的index。grid-column-end、grid-row-end的值也可以是span identifiers加上數字表示橫跨的軌道數。grid-column-start、grid-column-end 可簡寫成grid-column，第一個值為start，第二個值為end，中間使用/間隔。

grid對齊

當網格小於該容器時，要使網格對齊容器定位置，可在容器上使用justify-content和align-content。若要使item對齊所佔據的cell特定位置，可在容器上使用justify-items和align-items，若要使單個item對齊所佔據的cell特定位置，在該item上使用align-self和justify-self。normal為*-content系列和*-items系列的預設值，auto為*-self的預設值。這些屬性有些通用的值如:normal值在有設定網格欄寬值的狀況下，在*-content property中相當start，在*-item和*-self中相當stretch，但若是items有定義尺寸也將相當於start。(flex-start在grid裡同start)start值將使item對齊所佔據的網格或使網格對齊容器的起始邊。(flex-end在grid裡同start)end值將使item對齊所佔據的網格或使網格對齊容器的結束邊。center值將使item對齊所佔據的網格或使網格對齊容器的中心。stretch值在*-content系列 property中，當網格高度(row)或寬度(col)未定義，且網格高度或寬度低於容器時作用，而在*-items、*-self系列 property中，當item尺寸未定義，且item寬度或高度小於網格cell時作用。
